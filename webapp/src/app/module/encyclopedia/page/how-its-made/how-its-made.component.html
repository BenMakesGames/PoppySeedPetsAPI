Graphics

The vast majority of Poppy Seed Pets' graphics are SVGs - Scalable Vector Graphics - meaning you can zoom in as much as you want, and never see pixelation. Even cooler, it allows me to recolor parts of images on the fly, such as the pet graphics!

I primarily use Inkscape to make the graphics. Inkscape is an open source project, so if you're interested in messing around with vector-based art, you can get it for free!

For Poppy Seed Pets' few non-vector images (such as the news post headers), I've been using AI art generation tools, primarily AUTOMATIC1111/stable-diffusion-webui, and training models from civitai.com. Like Inkscape, these things are open source & free, however they don't come with easy-to-use installers. If opening up a command prompt and typing instructions doesn't scare you off, though, give 'em a try. It's a cool technology to play around with!

Website

There are several layers to the website...

Database

Most of the game's data (information about your pets, inventory, activity logs, etc) are stored in a relational database; in particular, a "flavor" of MySQL.

There's a whole family of MySQL and MySQL-compatible database engines, most of which are open-source. If you're looking to play with a database, yourself, PostgreSQL is open-source, and pretty easy to install on your own computer.

Application

A database only stores data. To do interesting things with that data, you gotta' do some programming to describe actions and behaviors to take on that data. This is everything from how a pet decides to "eat", and what it means to eat (an item is deleted, and a pet's stomach is filled), to moving items around, preparing recipes, etc.

For Poppy Seed Pets, I wrote this part - the application - in PHP. I'm also using many open-source libraries, perhaps most-notably Symfony. Symfony is an open-source framework that makes it easy to put a PHP program on the web, as a web API.

There are a lot of languages and frameworks for making websites; the main reason I picked PHP & Symfony is really just because it's what I was using a lot at work around the time I started Poppy Seed Pets. If I were to start Poppy Seed Pets again, from scratch, I'd almost certainly use ASP.NET Core and C#, instead.

Years ago, PHP was the no-brainer go-to for making web applications - there weren't many other, compelling, open-source options - but these days C# is really the better choice: though it still has a reputation of being closed-source and Windows-only, that all changed many years ago. And unlike PHP, C# has a huge team behind it, able to release interesting & useful updates every year.

Web Front-end

Web APIs aren't very user-friendly: they don't provide graphics and buttons and things, just a bunch of data describing things like how hungry your pet it, or what species it is. To interpret that data (and actually show an image of your pet, for example), I had to build a "front-end" - the thing your eyeballs and/or earholes are perceiving, right now!

As with web APIs, there are open-source frameworks that help creating such front-ends; for Poppy Seed Pets, I'm using Angular.

Angular is a JavaScript front-end framework that Google started a few years back. It's cousins-ish with other JS front-end frameworks like Vue and React + Redux. Purely anecdotally, from my experiences, React feels like the #1 choice for most web app developers, while Angular is a comfortable #2. But again, that's not based on any real research; just a feeling I get from googling around and talking to other developers. All these frameworks are pretty good, though, and a lot of what you choose to use comes down to personal preference.

If you'd like to experiment with making a web front-end, you don't need a web API, or database, or anything of the other stuff I've mentioned here; you can get started with any of the front-end frameworks I've mentioned, and all of them have very good getting-started tutorials (and, of course, there's always YouTube!)

As for why I chose Angular for Poppy Seed Pets: 1. compared to other web front-end frameworks, it's got a lot of stuff built-in, and I like that in a framework (this does mean it has a higher learning curve, however); 2. it uses TypeScript, a language by Microsoft that "transpiles" into JavaScript. TypeScript adds a lot of features to JS, such as strong typing (when you want it (which should be 99% of the time)), enums, generics, interfaces, and other syntactical niceties.

Cloud Hosting

Finally, I can code all of this on my personal computer, but once that's done, I have to actually put it somewhere on the internet, with a domain name, in order for y'all to access it! For that, Poppy Seed Pets is hosted by Amazon's Web Services: their RDS services for the database, EC2 + ALB for the API, S3 + CloudFront for the web app, and a few other services.

Microsoft's Azure has comparable services, but Amazon has been around longer, has more features, and their UI makes more sense to me. That said, Azure makes it easier to get started with some of their services for free, so if you're looking to play around with cloud hosting (and especially if you want to code with ASP.NET Core in C#, which Azure makes extra-easy to host), you might want to start with Azure. There's definitely a lot to learn with cloud hosting services, though! If you haven't used one before, be prepared to spend a lot of time scratching your head and watching YouTube tutorials.
